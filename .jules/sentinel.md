## 2024-05-24 - [CRITICAL] Hardcoded Firebase Credentials in Source Code
**Vulnerability:** Firebase `apiKey`, `appId`, and other configuration details were hardcoded directly into `src/firebase/config.ts` and committed to the repository. While some Firebase config is public, hardcoding it directly is risky if secrets accidentally slip in, limits environment flexibility, and is a poor security practice overall.
**Learning:** Configurations were baked directly into a code file instead of being injected via the environment, leading to the deployment of these values directly via source control. Relying on iteration for validation made the code less searchable and prone to obfuscation.
**Prevention:** All environment-specific variables, especially API keys and IDs, must be injected using `process.env.NEXT_PUBLIC_*` (or backend equivalents for secrets). Adding explicit, non-iterative checks (e.g., `if (!apiKey) throw new Error(...)`) ensures the application fails securely and predictably if misconfigured while keeping code easily searchable and maintainable. A `.env.example` must be provided to guide developers without committing actual values.
